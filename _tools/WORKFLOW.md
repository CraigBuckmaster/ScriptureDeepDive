# Content Authoring Workflow

## Pipeline Overview

```
Generator Script → save_chapter() → JSON → build_sqlite.py → scripture.db → OTA
```

All content is authored as Python dicts, saved as JSON, assembled into SQLite, and deployed to devices via over-the-air updates. No HTML is generated.

---

## Writing New Chapters

1. **Copy the template:**
   ```bash
   cp _tools/GENERATOR_TEMPLATE.py /tmp/gen_{book}_{range}.py
   ```

2. **Fill in scholarly content** — same format as always:
   - Section-level: `heb` (4-tuples), `ctx` (string), `cross` (2-tuples), `mac`/`calvin`/`netbible` (2-tuples)
   - Chapter-level: `lit` (rows, note), `themes` (10 scores, note)
   - Missing chapter panels are auto-generated by `auto_scholarly_json()`

3. **Run the generator:**
   ```bash
   python3 /tmp/gen_{book}_{range}.py
   ```

4. **Validate content:**
   ```bash
   python3 _tools/validate.py
   ```

5. **Build database:**
   ```bash
   python3 _tools/build_sqlite.py
   ```

6. **Verify database:**
   ```bash
   python3 _tools/validate_sqlite.py
   ```

7. **Commit:**
   ```bash
   git add content/ scripture.db && git commit -m "Add {book} {range}"
   git push origin master
   ```

8. **Deploy to devices:**
   ```bash
   eas update --branch production
   ```

## Adding a New Book

1. **Add to `config.py`:**
   - `BOOK_META['{book}']` — author text, VHL word lists
   - `COMMENTATOR_SCOPE` — add book-specific scholars

2. **Update `content/meta/books.json`:**
   - Set `is_live: true` for the book
   - Or run `python3 _tools/export_config.py` to regenerate

3. **Write chapters** using `GENERATOR_TEMPLATE.py`

4. **Follow steps 4–8 above** (validate → build → verify → commit → deploy)

---

## Adding a New Scholar

1. **Add to `config.py`:**
   - `SCHOLAR_REGISTRY` — (panel_key, scholar_key, label, css_suffix)
   - `COMMENTATOR_SCOPE` — scope: `'all'` or `['book1', 'book2']`

2. **Add bio page** in `_archive/commentators/{key}.html` (for extraction)

3. **Add to `commentators/scholar-data.js`** (or `content/meta/scholar-data.json` directly)

4. **Run `python3 _tools/export_config.py`** to regenerate `scholars.json`

5. Use the scholar key in generator scripts: `'{key}': [('ref', 'note')]`

## Tool Reference

| Tool | Purpose |
|------|---------|
| `save_chapter(book, ch, data)` | Write chapter JSON from generator data |
| `verse_range(start, end)` | Generate verse number list for sections |
| `auto_scholarly_json(data, book, ch)` | Auto-generate missing chapter panels |
| `validate.py` | Check all content JSON for schema/completeness |
| `build_sqlite.py` | Assemble content/ → scripture.db |
| `validate_sqlite.py` | Verify database integrity (51 checks) |
| `migrate_content.py` | One-time migration of HTML→JSON formats |
| `extract_to_json.py` | Extract chapter HTML→JSON (archive tool) |
| `convert_js_to_json.py` | Convert JS data files→JSON (archive tool) |
| `export_config.py` | Export Python config→JSON |

## Data Format Quick Reference

**Section panels** (inside each section dict):
```python
'heb': [('word', 'transliteration', 'gloss', 'paragraph about the word')]
'ctx': 'Historical context paragraph...'
'cross': [('Ref', 'Cross-reference note...')]
'mac': [('1:1', 'MacArthur note...')]        # same format for all scholars
```

**Chapter panels** (top-level in data dict):
```python
'lit': ([('vv.1-5', '1:1-5', 'Section title', True), ...], 'Structure note')
'themes': ([('Covenant',7), ('Judgment',8), ...], 'Theme note')  # 10 scores
```
